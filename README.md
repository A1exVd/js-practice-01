### Вопросы к заданию

**Почему в современном коде мы избегаем использование var?**

Переменные, Объявленные с var не имеют блочной области видимости, подвергаются поднятию, и их можно переопределять, что создаёт предпосылки для большего количество ошибок и сложности пониманию и отладки кода. Например, использование таких переменных в лупах или кондишинах засоряет область видимости, в которой эти конструкции используются, этими переменными.


**Что произойдет, если попытаться изменить значение переменной, объявленной через const?**

Возникнет ошибка времени выполнения, то есть код парсится интерпретатором правильно,
но во время выполнения происходит ошибка, а именно TypeError, которая возникает, когда мы пытаемся выполнить операцию
не предназначенную для этого типа.
Uncaught TypeError: Assignement to constant variable.

**Почему мы используем textContent вместо innerHTML для вывода простого текста?**

textContent используется для вставки текста в html элемент, что в данном случае 
является наиболее целесообразным, в то время как innerHtml позволяет вставлять html элементы 
внутрь другого html элемента,то есть содержимое innerHtml парсится и отображается 
как html элементы, что можем повысить риск XSS атак. Кроме того, использование 
innerHtml практически всегда триггерит рефлоу, что может влиять на производительность
и замедлить отображение страницы. 


**В чем разница между let и const? Когда какую переменную использовать?**

Основное отличие между const и let заключается в том, что переменную объявленную с const 
нельзя переопределить, то есть нельзя присвоить ей новое значение после ее инициализации
также нельзя объявить переменную с const без значения. let используется, когда мы точно
знаем, что будем присваиваить этой переменной новое значение. Например, в for in лупах или 
в качестве булеан флага. В случаях, когда переопределение не требуется используем const.

**В чем разница между addEventListener и написанием события прямо в HTML-теге (например, onclick "...")?**
**Почему addEventListener считается более правильныи подходом, чем inline-события в HTML?**

Использование addEventListener позволяет не засорять html код js кодом, делая html файл более тяжелым.
Также опционный параметр useCapture метода addEventListener позволяет определить на фазе всплытия или поднятия 
сработает обработчик, в то время как inline-обработчики всегда срабатывают на фазе всплытия.
Используя addEventListener мы можем добавить несколько обработчиков к событию, а при использовании инлайновых
выполняться будет только один обработчик (последний).
